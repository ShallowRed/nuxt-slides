# Instructions pour la Création de Contenu de Présentation Nuxt + Reveal.js

## Objectif
Générer du contenu structuré au format Markdown (MDC) pour des présentations reveal.js dans un système Nuxt.

## Format de Fichier

### Extension
`.md` (Markdown)

### Structure Complète
```markdown
---
title: "Titre de la Présentation"
theme: dsfr
transition: slide
center: true
backgrounds:
  h1: "/backgrounds/titre.jpg"
  h2: "/backgrounds/section.jpg"
---

# Titre Principal
Sous-titre ou description courte

## Première Section
Contenu de la première diapositive

### Sous-section 1
Détails ou points spécifiques

### Sous-section 2
Plus de détails

## Deuxième Section
Nouveau sujet principal

## Règles de Structure

### Hiérarchie des Titres

1. **H1 (`#`)** - Diapositive de titre
   - UN SEUL par présentation
   - Premier titre de la présentation
   - Tout le contenu jusqu'au premier H2 est inclus
   - Classe CSS: `slide-h1`

2. **H2 (`##`)** - Diapositives horizontales
   - Sépare les sections principales
   - Navigation: flèches gauche/droite
   - Classe CSS: `slide-h2`

3. **H3 (`###`)** - Diapositives verticales
   - Sous-sections imbriquées dans un H2
   - Navigation: flèches haut/bas
   - Classe CSS: `slide-h3`

4. **HR (`---`)** - Séparateur alternatif
   - Équivalent à H2 sans titre
   - Crée une nouvelle diapositive horizontale

### Métadonnées (Frontmatter)

```yaml
---
title: "Titre Affiché"                    # Obligatoire pour l'index
lang: fr                                   # Langue (optionnel, ex: fr, en)
theme: dsfr                                # Options: dsfr, minimal, lee, dsfr-alt
transition: slide                          # Options: slide, fade, convex, zoom, none
center: true                               # Centrage vertical: true/false
backgrounds:                               # Images de fond (optionnel)
  h1: "/backgrounds/image-titre.jpg"
  h2: "/backgrounds/image-section.jpg"
  h3: "/backgrounds/image-detail.jpg"
  default: "/backgrounds/image-defaut.jpg" # Fond par défaut pour toutes les autres diapos
parser: separator                          # Optionnel: mode "separator" (utilise ---- au lieu de ## pour les slides horizontales)
unlisted: true                             # Masquer de l'index public (optionnel)
reveal:                                    # Surcharge de la config Reveal.js (optionnel)
  slideNumber: true
  width: 1200
  height: 800
---
```

#### Mode `parser: separator`
Quand `parser: separator` est activé, les diapositives horizontales sont séparées par `----` (quatre tirets) au lieu des titres `##`. Les titres `##` deviennent alors de simples titres de section dans la diapositive.

```markdown
# Titre de la présentation

----

## Première section

----

## Deuxième section
```

## Composants MDC Disponibles

### 1. Columns (Colonnes)

**Mode auto (split sur les titres H3)** — le plus courant :
```markdown
::Columns
### Colonne Gauche
Contenu de la première colonne

### Colonne Droite  
Contenu de la seconde colonne
::
```

**Mode explicite avec `::column`** — pour un contrôle fin (utiliser `:::` pour l'encadrant) :
```markdown
:::Columns{count=3 size="sm"}
::column
Contenu de la colonne 1

::column
Contenu de la colonne 2

::column
Contenu de la colonne 3
:::
```

**Colonne à largeur variable** (`span` ou `ratio`) :
```markdown
:::Columns
::column{span=2}
Colonne large (2/3)

::column{span=1}
Colonne étroite (1/3)
:::
```

Paramètres de `Columns` :
- **count**: nombre de colonnes pour le mode auto (2 par défaut)
- **size**: taille du texte — `xs`, `sm`, `md` (défaut), `lg`, `xl`

Paramètres de `::column` :
- **span**: poids relatif (ex: `span=2` vs `span=1` → 2/3 + 1/3)
- **ratio**: ratio explicite (ex: `ratio="2/3"`)

Les alias `::TwoColumns` et `::ThreeColumns` fonctionnent aussi (rétrocompatibilité).

### 2. Quote (Citation)
```markdown
::Quote{author="Nom Auteur" role="Titre/Rôle" avatar="/images/photo.jpg"}
Texte de la citation
::
```
- **author**: nom de l'auteur (optionnel)
- **role**: titre ou rôle (optionnel)
- **avatar**: URL de la photo de l'auteur (optionnel)

### 3. Image (Image)
```markdown
::Image{src="/images/photo.jpg" alt="Description" preview}
::
```
- **preview**: active la lightbox Reveal.js au clic. Valeur booléenne ou URL d'une vignette distincte
- **width**, **height**: dimensions optionnelles
- **fit**: `contain` (défaut), `cover`, `fill`, `none`, `scale-down`
- **position**: position de l'image (défaut: `center`)
- **rounded**: coins arrondis (défaut: `true`)
- **shadow**: ombre portée (défaut: `true`)

### 4. Layout média (diapositive à deux colonnes)
```markdown
### Titre de la diapositive
:layout{name="media-right" src="https://example.com" type="iframe" lightbox="true"}

Contenu textuel à gauche, média à droite.
```
- **name**: 
  - `"media-right"` / `"media-left"` — partage équilibré (50/50)
  - `"media-right-wide"` / `"media-left-wide"` — média plus large
- **src**: URL de l'iframe ou chemin de l'image
- **type**: `"iframe"` ou `"image"`
- **lightbox**: `"true"` pour activer l'ouverture en plein écran au clic (iframe uniquement)
- **fit**: `"contain"` ou `"cover"` pour les images (contrôle l'object-fit)
- **title** / **alt**: attributs optionnels pour iframe/image

### 5. FullScreenImage (Image Plein Écran)
```markdown
::FullScreenImage{src="/chemin/image.jpg" overlay="0.4" position="center"}
# Titre sur l'image
Contenu avec fond d'image
::
```
- **overlay**: opacité du fond noir (0-1)
- **position**: "center", "top", "bottom", "left", "right"

### 6. Mermaid (Diagrammes)
```markdown
::Mermaid
graph TD
    A[Début] --> B[Étape 1]
    B --> C[Étape 2]
    C --> D[Fin]
::
```

### 7. Iframe (Contenu Intégré)
```markdown
::Iframe{src="https://example.com" height="500px"}
::
```
- **height**: hauteur de l'iframe (défaut: `500px`)
- **fullwidth**: prend toute la largeur (défaut: `true`)
- **lightbox**: active la lightbox Reveal.js (clic pour agrandir) (défaut: `false`)

### 8. PreviewLink (Aperçu de Lien)
```markdown
::PreviewLink{href="https://example.com" label="Voir le site"}
::
```
- **href**: URL à ouvrir (obligatoire)
- **label**: texte du bouton (optionnel, affiche l'URL si absent)
- Le contenu slot remplace le label si présent

### 9. Icônes Inline
```markdown
:i{name="ri-home-line"} Accueil
:i{name="ri-settings-line"} Paramètres
:icon{name="ri-heart-line"} Alternatif
```
- `:i{}` et `:icon{}` sont équivalents
- **size**: taille optionnelle (ex: `size="1.5em"`)
- Utiliser les noms Remix Icons: `ri-nom-de-l-icone` (voir [remixicon.com](https://remixicon.com))

### 10. Annotations de Diapositive (Inline)
```markdown
:slide-background{image="/backgrounds/custom.jpg"}
:pretitle{text="Chapitre 1"}
:subtitle{text="Sous-titre de la diapositive"}
:quicklink{text="Aller à la conclusion" href="#conclusion"}
```
- Placés dans le contenu d'une diapositive, avant le texte principal
- Ne génèrent pas de contenu visible dans le flux markdown
- **:slide-background** — surcharge l'image de fond de la diapositive (prop `image`, non `src`)
- **:pretitle** — texte affiché au-dessus du titre
- **:subtitle** — texte affiché sous le titre
- **:quicklink** — lien de navigation épinglé en bas de diapositive (ancre interne)

### 11. Highlight (Mise en avant)
```markdown
::Highlight{size="lg" variant="accent" center}
Message clé à mettre en avant
::
```
- **size**: `sm`, `md` (défaut), `lg`, `xl`
- **variant**: `default` (hérite couleur slide), `accent` (bordure bleue + fond clair), `contrast` (fond sombre, texte blanc)
- **center**: centre le texte horizontalement (optionnel)

## Syntaxe Markdown Standard

### Formatage Texte
```markdown
**texte en gras**
*texte en italique*
`code inline`
~~texte barré~~
```

### Listes
```markdown
- Liste non ordonnée
- Item 2
  - Sous-item

1. Liste ordonnée
2. Item 2
   1. Sous-item
```

### Code
````markdown
```javascript
const exemple = 'Code avec coloration syntaxique'
console.log(exemple)
```
````

### Liens
```markdown
[Texte du lien](https://example.com)
```

### Images
```markdown
![Texte alternatif](/images/photo.jpg)
```

### Citations
```markdown
> Citation importante
> Sur plusieurs lignes
```

## Bonnes Pratiques

### Contenu des Diapositives
1. **Concision**: Maximum 5-7 points par diapositive
2. **Phrases courtes**: Privilégier les énoncés directs
3. **Mots-clés**: Mettre en avant les concepts importants
4. **Hiérarchie visuelle**: Utiliser les niveaux de titres appropriés

### Organisation
1. **Introduction** (H1): Titre, contexte, objectifs
2. **Développement** (H2): Sections thématiques principales
3. **Détails** (H3): Sous-points pour approfondir
4. **Conclusion** (H2): Synthèse, appel à l'action

### Images et Médias
- Placer dans `public/images/`
- Référencer: `![](/images/nom-fichier.jpg)`
- Formats: JPG, PNG, SVG, WebP
- Optimiser: < 500KB par image

### Thèmes

#### DSFR (Système de Design de l'État Français)
- Style officiel gouvernemental
- Couleurs: Bleu (#000091), gris
- Police: Marianne
- Usage: Présentations institutionnelles

#### Minimal
- Design épuré et moderne
- Couleurs: Tons clairs
- Police: Helvetica
- Usage: Présentations simples, techniques

## Checklist de Validation

Avant de finaliser le contenu, vérifier:

- [ ] Frontmatter complet et valide
- [ ] UN SEUL H1 au début
- [ ] H2 pour sections principales
- [ ] H3 uniquement sous H2 (pour verticales)
- [ ] Pas plus de 5-7 points par diapositive
- [ ] Code formaté avec triple backticks
- [ ] Images optimisées et chemins corrects
- [ ] Composants MDC bien fermés (`::`)
- [ ] Pas de lignes vides excessives
- [ ] Ponctuation et orthographe vérifiées

## Conseils pour un Contenu Efficace

### Rythme
- Alterner contenu dense et diapositives simples
- Prévoir 2-3 minutes par diapositive H2
- 30 secondes par diapositive H3

### Visuel
- Privilégier les listes à puces aux paragraphes
- Mettre en **gras** les messages clés
- Ajouter des images pour illustrer concepts complexes

### Narration
- Commencer par le "pourquoi"
- Développer le "comment"
- Conclure avec le "quoi faire"

### Interactivité
- Poser des questions rhétoriques
- Inclure des exemples concrets
- Prévoir des pauses de réflexion

## Erreurs à Éviter

1. ❌ Plusieurs H1 dans la présentation
2. ❌ H3 sans H2 parent
3. ❌ Paragraphes trop longs (> 3 lignes)
4. ❌ Plus de 7 points par diapositive
5. ❌ Images non optimisées (> 500KB)
6. ❌ Composants MDC mal fermés
7. ❌ Oublier le frontmatter
8. ❌ Utiliser des chemins relatifs pour images

## Commandes Utiles

Une fois le contenu créé:

1. Placer le fichier dans `presentations/nom-fichier.md`
2. Compiler les thèmes: `pnpm build:themes`
3. Lancer le serveur: `pnpm dev`
4. Accéder: `http://localhost:3000`
5. Visualiser: `http://localhost:3000/slides/nom-fichier`

## Navigation dans la Présentation

- `→` / `←` : Diapositives horizontales (H2)
- `↓` / `↑` : Diapositives verticales (H3)
- `Espace` : Diapositive suivante
- `Esc` : Vue d'ensemble
- `F` : Plein écran
- `S` : Notes de présentateur (si activées)
